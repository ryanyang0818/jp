# TESTING.md

## 測試方式概述

本專案為純靜態 HTML，**無自動化測試框架**（無 Jest、Vitest、Playwright 等）。所有驗證透過瀏覽器手動測試進行。

---

## 測試環境要求

### 建議瀏覽器

| 瀏覽器 | 平台 | TTS 支援 | boundary 事件 | 備註 |
|--------|------|----------|----------------|------|
| Chrome 最新版 | Windows | ✅ | ✅ 較完整 | **首選測試環境** |
| Edge 最新版 | Windows | ✅ | ✅ | 同 Chromium 核心 |
| Chrome | macOS | ✅ | ✅ | 可能與 Windows 行為略異 |
| Safari | macOS | ✅ | ⚠️ 不穩定 | boundary 事件行為不同 |
| Firefox | 任何 | ⚠️ 部分 | ❌ 通常不觸發 | 不推薦測試 TTS 功能 |
| Chrome | Android | ✅ | ⚠️ | 行動端語音引擎不同 |

### 啟動方式

```bash
# 使用本地伺服器（避免 CORS 問題，確保 vocabulary.json 能正常載入）
python -m http.server 8080
# 開啟 http://localhost:8080/index2.html
```

## 各頁面手動測試清單

### index2.html

#### 基本功能
- [ ] 頁面載入後，顯示第 1 張卡片（うえ / 上 / ue）
- [ ] 進度顯示「0 / 10 已翻開」
- [ ] 10 個點點指示器全部顯示（灰色）
- [ ] 第 1 個點點為藍色（active）

#### 翻卡互動
- [ ] 點擊卡片 → 卡片向右翻至背面（顯示「上面・上方」+ 例句）
- [ ] 再次點擊 → 翻回正面
- [ ] 按 `Space` → 向右翻轉
- [ ] 按 `↓` → 向下翻轉（rotateX）
- [ ] 按 `↑` → **無反應**（已停用，保留另作他用）
- [ ] 翻面後按不同方向鍵 → 切換翻轉方向，動畫不跳幀

#### 導覽
- [ ] 按 `→` 或點擊「下一張」→ 跳至第 2 張（した / 下 / shita）
- [ ] 按 `←` 或點擊「上一張」→ 返回第 1 張
- [ ] 第 10 張按「下一張」→ 循環回第 1 張
- [ ] 第 1 張按「上一張」→ 循環至第 10 張
- [ ] 點擊任意點點 → 直接跳至對應卡片

#### 進度追蹤
- [ ] 翻開 3 張後，進度顯示「3 / 10 已翻開」，進度條寬度 30%
- [ ] 已翻卡片的點點顯示綠色
- [ ] 切換回已翻卡片，重複翻轉不重複計入進度

#### TTS 自動朗讀
- [ ] 右翻卡片 → 自動朗讀平假名讀音（如「うえ」）
- [ ] 下翻卡片（↓）→ 自動朗讀日文例句（如「机の上に本があります。」）
- [ ] 翻回正面 → 無聲音
- [ ] 播放中切換至下一張並翻牌 → 舊語音停止，新語音播放

#### 語速滑桿
- [ ] Header 顯示語速滑桿，預設值 1.0x
- [ ] 拖曳至 0.5x → 顯示「0.5x」，下次翻牌語速明顯變慢
- [ ] 拖曳至 1.5x → 顯示「1.5x」，下次翻牌語速明顯變快
- [ ] 拖曳時數值即時更新（不需放開滑鼠）

#### 逐字高亮
- [ ] 翻牌後，`#exJp` 的例句文字逐字變紅（由左至右）
- [ ] 高亮速度與語速滑桿連動（1.5x 時高亮比 0.5x 快）
- [ ] 切換卡片後，上一張的高亮序列立即停止（無殘留紅字閃爍）

#### Fallback 測試
- [ ] 使用 `file://` 協定開啟（或阻擋 vocabulary.json）→ 頁面仍正常顯示 10 張卡片

## 瀏覽器相容性測試

### CSS 3D Transform

```javascript
// 在瀏覽器 Console 執行，確認支援
CSS.supports('transform-style', 'preserve-3d');  // true
CSS.supports('backface-visibility', 'hidden');   // true
```

### Web Speech API

```javascript
// 在瀏覽器 Console 執行
'speechSynthesis' in window;  // true = 支援

// 確認有日語語音可用
speechSynthesis.getVoices().filter(v => v.lang.startsWith('ja'));
// 應返回至少一個語音（空陣列 = 無日語 TTS）
```

**注意**：`getVoices()` 可能在頁面載入後需要幾秒才能取得完整清單。如果立即呼叫返回空陣列，改用：

```javascript
speechSynthesis.onvoiceschanged = () => {
  console.log(speechSynthesis.getVoices().filter(v => v.lang.startsWith('ja')));
};
```

## 常見問題排查

| 問題 | 可能原因 | 解法 |
|------|----------|------|
| index2.html 無法載入詞彙 | 使用 `file://` 協定，CORS 阻擋 fetch | 改用本地伺服器，或確認 FALLBACK 是否正常 |
| TTS 無聲音 | 瀏覽器或 OS 無日語 TTS 語音 | 安裝日語語音包（Windows：語言設定；macOS：系統偏好設定 > 語音）|
| TTS 無聲音（iOS Safari） | 需要使用者互動後才能播放音訊 | 先點擊頁面任意地方，再點 🔊 |
| 3D 翻轉動畫不順暢 | 低效能裝置或舊瀏覽器 | 確認 Chrome/Edge 最新版；行動裝置可能有 CSS 3D 限制 |
| boundary 事件不觸發（測試2.html） | 瀏覽器 TTS 引擎不支援 | 換 Chrome on Windows/macOS 測試 |
| 高亮與語音不同步（特效版） | setTimeout 基準值（200ms/char）不準 | 修改 `msPerChar = 200 / rate` 的基準值；參考 FEATURES.md 待驗證項目 |

## 新增測試的步驟

本專案無測試框架，新增測試等同於在本文件加入新的測試項目：

1. 確定要測試的功能（對應 FEATURES.md 中的功能編號）
2. 在本文件對應頁面的測試清單下新增 `- [ ] 測試描述`
3. 描述格式：`操作 → 預期結果`（例如：`點擊 🔊 → 播放日語語音`）
4. 執行後打勾：`- [x] 測試描述`
